# AGENTS.md

## Project Overview

Single Spring Boot module (`simple-service`) secured as an OAuth2 Resource Server via **Keycloak**, with users federated from **OpenLDAP**. Primarily a demo project.

- Java 25, Spring Boot 4.0.6, Maven Wrapper (`./mvnw`)
- Root `pom.xml` is `packaging=pom`; only real app module is `simple-service/`

---

## Developer Commands

### Build & Package
```bash
./mvnw clean package          # compile + test + spotless:check (verify phase)
./mvnw clean test             # tests only — does NOT run Spotless check
./mvnw spotless:check         # check formatting only
./mvnw spotless:apply         # auto-fix formatting
```

> **Spotless runs at `verify`, not `test`.** `./mvnw test` will NOT fail on formatting issues. Only `verify` or later phases enforce it.

### Run App (dev)
```bash
./mvnw clean spring-boot:run --projects simple-service \
  -Dspring-boot.run.jvmArguments="-Dserver.port=9080"
```

Swagger UI: `http://localhost:9080/swagger-ui.html`

### Single Test / Class
```bash
./mvnw test --projects simple-service -Dtest=SimpleServiceControllerTests
./mvnw test --projects simple-service -Dtest=SimpleServiceControllerTests#shouldReturnPublicString
```

### Docker Images
```bash
./build-docker-images.sh           # JVM image (skips tests automatically)
./build-docker-images.sh native    # GraalVM Native image
./remove-docker-images.sh
```

## Infrastructure Setup (Required to Run the App)

All infra runs via Docker Compose. Required setup order:

```bash
docker compose up -d              # starts postgres, keycloak, openldap, phpldapadmin
./import-openldap-users.sh        # seeds LDAP with 4 users
./init-keycloak.sh                # creates realm, client, roles, LDAP federation
```

`init-keycloak.sh` **prints `SIMPLE_SERVICE_CLIENT_SECRET` at the end** — this value is regenerated each time and must be captured. Requires `jq` on the host.

```bash
docker compose down -v            # tear down including volumes
```

## Testing

- **No Docker or Keycloak required for tests.** All tests use `@WebMvcTest` + `MockMvc` + Spring Security `jwt()` test support.
- `JwtDecoder` and `JwtAuthenticationTokenConverter` are mocked with `@MockitoBean`.
- `@Import(SecurityConfig.class)` is on the test class — full security filter chain is active.
- JWT mocks without `USER` role return `403`, not `200`.

## Non-Obvious Gotchas

- **Docker network name** when running the app container manually: `springboot-keycloak-openldap_default`
- **`SIMPLE_SERVICE_CLIENT_SECRET`** has no static default — regenerated by Keycloak on each env rebuild; always re-run `./init-keycloak.sh` and capture it
- **Token expiry is 5 minutes** (Keycloak default) — relevant for manual curl testing
- JWT role extraction uses a custom converter reading from the `simple-service` client in the Keycloak token; principal attribute is `preferred_username`
- No CI workflows exist in this repo
